Optional dispatchEventsOnWindow

dispatchEventsOnWindow?: boolean

If you want to listen to the events from inside your application, the events can also be dispatched in the browser as custom events by vue-store-oidc (prefixed with OIDC_BBROWSER_EVENT_PREFIX:). If you want this, you can pass dispatchEventsOnWindow: true as a storeSetting.

Optional isAuthenticatedBy

isAuthenticatedBy?: "access_token" | "id_token"

By default this package binds access to if the user has a valid id_token. You can change this with the isAuthenticatedBy property on the storeSetting argument so access is bound to a valid access_token in stead: 

const oidcStore = new PiniaOidcStore(
 oidcSettings,
 { 
   dispatchEventsOnWindow: true,  
   isAuthenticatedBy: 'access_token'
 }
)

Optional publicRoutePaths

publicRoutePaths?: string[]

If you want to have some of your routes public, you can specify those as publicRoutePaths in the storeSetting argument:

const oidcStore = new PiniaOidcStore(
 oidcSettings,
 { 
   dispatchEventsOnWindow: true,  
   publicRoutePaths: ['/', '/about-us']
 }
)

Optional routeBase

routeBase?: string

If you use vue-router with a router base other than the default (/), you will want to pass your route as routeBase in the storeSetting argument:

 oidcSettings,

   dispatchEventsOnWindow: true,
